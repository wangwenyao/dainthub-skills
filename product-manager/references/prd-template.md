# PRD模板

产品需求文档标准模板结构。

---

## 一、自适应文档结构

**根据端划分自动选择文档结构**：

| 端数量 | 文档结构 | 适用场景 |
|--------|----------|----------|
| ≥ 2 个端 | **三层结构** | 多端产品（小程序+管理后台等） |
| = 1 个端 + 模块 ≥ 3 | **两层结构** | 单端多模块产品 |
| = 1 个端 + 模块 ≤ 2 | **单层结构** | 简单单端产品 |

**结构选择流程**：
```
端划分确认（需求分析输出）
    ├── 端数量 ≥ 2 → 三层结构
    ├── 端数量 = 1 → 检查模块数量
    │   ├── 模块 ≥ 3 → 两层结构
    │   └── 模块 ≤ 2 → 单层结构
    └── 无端划分 → 默认两层结构（管理后台）
```

| 端名称 | 主要角色 | 核心功能 | 独立PRD |

> 各端共用的业务实体定义（精简版）

## 五、通用功能模板库

> **⚠️ 重要说明**：此模板库内容应复制到总览PRD第五章（"共享通用功能模板库"），而非放在skill内部。
> - 开发人员阅读的是PRD文档，不是skill文件
> - 总览PRD第五章是模板库的正确位置
> - 各端PRD引用格式：`@总览PRD#5.X [模板ID] [模板名称]`

**目的**：减少重复内容，引用模板 + 差异标注。
