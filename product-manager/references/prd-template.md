# PRD模板

产品需求文档标准模板结构。

---

## 一、文档信息

```markdown
# 产品需求文档 (PRD)

| 文档版本 | 1.0 |
|---------|-----|
| 编写日期 | YYYY-MM-DD |
| 文档状态 | 草稿/评审中/已确认 |

## 修订记录
| 版本 | 日期 | 修订人 | 修订内容 |
|------|------|--------|----------|
| 1.0 | YYYY-MM-DD | 姓名 | 初始版本 |
```

### 5. 用户旅程

```markdown
## 五、用户旅程
> 用户视角的流程路径，描述用户从入口到完成目标的步骤

### 5.1 核心用户流程路径
用户从进入系统到完成核心目标的页面路径，如：
首页 → 商品列表 → 商品详情 → 加入购物车 → 结算 → 支付 → 完成

### 5.2 异常路径
用户操作失败时的路径，如：
库存不足 → 返回商品详情 → 选择其他商品
```

## 五、章节顺序说明

| 章节 | 视角 | 说明 |
|------|------|------|
| 一至三 | 产品视角 | 愿景、目标、范围 |
| 四 | 角色视角 | 谁参与系统 |
| 五 | 用户视角 | 用户如何操作系统 |
| 六 | 业务视角 | 业务如何流转 |
| 七 | 业务视角 | 业务活动和规则 |
| 八 | 系统视角 | 系统功能设计 |
| 九至十 | 技术视角 | 数据模型、非功能需求 |

**顺序逻辑**：用户需求 → 业务流程 → 系统功能设计
