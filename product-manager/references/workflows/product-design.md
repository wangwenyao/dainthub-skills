# 产品设计流程

**何时使用**: 设计产品方案、规划功能、定义用户体验时

---

## 流程步骤

```
1. 设计思维流程（Design Thinking）
   ├── 同理心（Empathize）→ 用户研究、访谈观察
   ├── 定义（Define）→ 问题陈述、痛点聚焦
   ├── 构思（Ideate）→ 头脑风暴、方案生成
   ├── 原型（Prototype）→ 快速原型、低保真测试
   ├── 测试（Test）→ 用户测试、反馈收集
   └── 详细方法 → @../product-design.md#Design-Thinking

2. 双钻模型
   ├── 第一钻石（发现问题）
   │   ├── Discover（发散）→ 研究探索
   │   └── Define（收敛）→ 问题定义
   ├── 第二钻石（解决问题）
   │   ├── Develop（发散）→ 方案开发
   │   └── Deliver（收敛）→ 方案交付
   └── 详细方法 → @../product-design.md#双钻模型

3. 用户体验五要素
   ├── 战略层 → 用户需求、产品目标
   ├── 范围层 → 功能规格、内容需求
   ├── 结构层 → 信息架构、交互流程
   ├── 骨架层 → 信息设计、导航设计
   ├── 表现层 → 视觉设计、感官体验
   └── 详细方法 → @../product-design.md#UX五要素

4. 输出
   └── 产品设计方案（功能定义、流程设计、原型草图）
```

---

## ⚠️ 强制输出映射（MANDATORY）

**UX 五要素输出必须映射到 PRD 章节**：

| UX 要素层 | 必须输出的内容 | 对应 PRD 章节 |
|-----------|---------------|---------------|
| **战略层** | 商业目标 + 用户核心需求 + 成功指标 | 一、执行摘要 + 二、成功标准 |
| **范围层** | In Scope / Out of Scope + MVP 功能筛选矩阵 | 三、产品范围 |
| **结构层** | 信息架构树 + 核心流程路径 | 五、用户旅程 + 八、功能菜单层级 |
| **骨架层** | 关键页面布局说明（非线框图，是布局决策） | 六、业务流程 + 线框图设计说明 |

**执行失败信号**：
- 产品设计流程执行后，上述 4 层均无对应产出 → **视为流程未实质执行**
- 战略层无成功指标 → **视为流程未实质执行**
- 范围层无 MVP 矩阵 → **视为流程未实质执行**
