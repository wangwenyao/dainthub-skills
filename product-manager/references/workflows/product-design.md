# 产品设计流程

**何时使用**: 设计产品方案、规划功能、定义用户体验时

---

## 流程步骤

```
1. 设计思维流程（Design Thinking）
   ├── 同理心（Empathize）→ 用户研究、访谈观察
   ├── 定义（Define）→ 问题陈述、痛点聚焦
   ├── 构思（Ideate）→ 头脑风暴、方案生成
   ├── 原型（Prototype）→ 快速原型、低保真测试
   ├── 测试（Test）→ 用户测试、反馈收集
   └── 详细方法 → @../product-design.md#Design-Thinking

2. 双钻模型
   ├── 第一钻石（发现问题）
   │   ├── Discover（发散）→ 研究探索
   │   └── Define（收敛）→ 问题定义
   ├── 第二钻石（解决问题）
   │   ├── Develop（发散）→ 方案开发
   │   └── Deliver（收敛）→ 方案交付
   └── 详细方法 → @../product-design.md#双钻模型

3. 用户体验五要素
   ├── 战略层 → 用户需求、产品目标
   ├── 范围层 → 功能规格、内容需求
   ├── 结构层 → 信息架构、交互流程
   ├── 骨架层 → 信息设计、导航设计
   ├── 表现层 → 视觉设计、感官体验
   └── 详细方法 → @../product-design.md#UX五要素

4. 输出
   └── 产品设计方案（功能定义、流程设计、原型草图）
```
