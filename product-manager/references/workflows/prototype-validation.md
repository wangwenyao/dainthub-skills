# 原型走查验证流程

**何时使用**: PRD 编写完成后、进入开发前，通过线框图走查验证需求闭环。

---

## ⚠️ 执行约束（MANDATORY）

**必须遵守以下规则，违反即为流程执行失败**：

| 规则 | 说明 |
|------|------|
| **必须有框图** | 走查基于线框图进行，无图不走查 |
| **必须覆盖核心流程** | 至少走完所有 P0 业务流程 |
| **必须记录问题** | 发现的每个问题必须记录，不得口头带过 |

---

## 输入与输出

| 输入 | 来源 |
|------|------|
| PRD 文档 | 流程三：PRD编写 输出 |
| 线框图 HTML | 线框图生成流程 输出 |
| 三级功能点清单 | PRD 第八章 |

| 输出 | 说明 |
|------|------|
| 走查问题清单 | 每个问题的描述、位置、严重等级 |
| 走查结论 | 通过 / 有条件通过 / 不通过 |
| PRD 修订建议 | 需要修改的章节和内容 |

---

## 走查方式

### 方式一：线框图走查（产品经理 + 业务方）

**目标**：验证页面结构和交互流程是否符合业务操作习惯。

**步骤**：
1. 打开线框图 HTML 文件
2. 从业务方实际工作入口开始，模拟完整操作路径
3. 检查每个页面的布局是否合理、字段是否完整
4. 检查交互跳转是否符合业务逻辑
5. 记录发现的问题

### 方式二：流程模拟（产品经理 + 业务方 + 技术）

**目标**：验证业务流程闭环，识别技术风险。

**步骤**：
1. 选取一个核心业务流程（如"订单创建→审批→执行→完成"）
2. 从触发者开始，逐步走完整个流程
3. 检查每个步骤是否有对应的功能支撑
4. 检查异常路径是否有处理方案
5. 检查权限矩阵是否覆盖所有操作
6. 技术侧评估实现可行性

### 方式三：功能清单评审（产品经理 + 技术经理）

**目标**：验证功能覆盖完整性。

**步骤**：
1. 对照三级功能点清单
2. 检查每个业务活动是否有对应功能点
3. 检查每个功能点是否有对应页面/组件
4. 检查权限矩阵是否完整
5. 检查优先级分布是否合理

## 走查检查清单

### 核心流程检查

```
☐ 每个 P0 业务流程是否都能从线框图中完整走完？
☐ 流程起点和终点是否明确？
☐ 流程中每个步骤是否有对应的页面/组件？
☐ 页面之间的跳转路径是否清晰？
☐ 是否存在流程断点（走到某一步没有后续页面）？
```

### 异常场景检查

```
☐ 空数据状态是否有设计（首次使用无数据）？
☐ 加载状态是否有反馈（接口慢时的提示）？
☐ 错误状态是否有处理（接口失败时的降级）？
☐ 权限不足时是否有提示？
☐ 并发冲突是否有考虑（多人同时编辑）？
```

### 功能覆盖检查

```
☐ 每个业务活动是否有对应功能点？
☐ 每个功能点是否有对应页面/组件？
☐ CRUD 操作是否完整？
☐ 搜索/筛选/排序/分页是否覆盖？
☐ 批量操作是否覆盖？
☐ 导入/导出是否覆盖？
```

### 权限覆盖检查

```
☐ 每个操作是否有明确的权限定义？
☐ 不同角色的操作范围是否清晰？
☐ 是否存在越权操作风险？
☐ 数据隔离规则是否定义（如只能看本人数据）？
```

## 问题记录格式

```markdown
### 走查问题清单

| # | 问题类型 | 问题描述 | 所在位置 | 严重等级 | 建议方案 |
|---|---------|---------|---------|---------|---------|
| 1 | 流程断点 | 审批通过后没有通知申请人的页面 | 审批流程第3步 | P0 | 增加审批结果通知 |
| 2 | 功能缺失 | 缺少批量导入功能 | 商品管理模块 | P1 | 增加导入按钮和弹窗 |
| 3 | 字段遗漏 | 新增表单缺少"供应商"字段 | P002 表单弹窗 | P1 | 补充供应商下拉选择 |
```

**严重等级定义**：

| 等级 | 定义 | 处理要求 |
|------|------|---------|
| **P0-阻塞** | 核心流程断点，无法完成业务操作 | 必须修复后才能进入开发 |
| **P1-重要** | 功能缺失或字段遗漏，影响使用体验 | 建议修复，可带条件进入开发 |
| **P2-建议** | 优化建议，不影响核心功能 | 可后续迭代处理 |

## 走查结论

### 结论判定

| 结论 | 条件 | 后续行动 |
|------|------|---------|
| **通过** | 无 P0 问题，P1 ≤ 3 个 | 直接进入开发 |
| **有条件通过** | 无 P0 问题，P1 > 3 个 | 修复 P1 问题后进入开发 |
| **不通过** | 存在 P0 问题 | 修复 P0 问题后重新走查 |
