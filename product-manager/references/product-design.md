# 产品设计方法论

产品设计核心方法论，从设计思维到UX五要素。

---

## 一、Design Thinking 设计思维

### 1.1 五阶段模型

```
同理心 → 定义 → 构思 → 原型 → 测试
```

| 阶段 | 目标 | 核心活动 |
|------|------|----------|
| 同理心 | 深入理解用户 | 用户访谈、同理心地图 |
| 定义 | 聚焦核心问题 | POV陈述、HMW问题 |
| 构思 | 生成解决方案 | 头脑风暴、SCAMPER |
| 原型 | 快速验证想法 | 低保真原型 |
| 测试 | 获取用户反馈 | 可用性测试 |

### 1.2 POV陈述格式

```
[用户] 需要 [需求]，因为 [洞察]
```

### 1.3 HMW问题格式

```
How Might We [问题]？
```

转换规则：将POV中的"因为"部分转化为HMW问题。

### 1.4 同理心地图模板

```
┌─────────────┬─────────────┐
│   想/感受   │     听      │
├─────────────┤             │
│     看      │             │
│             ├─────────────┤
│             │   说/做     │
├─────────────┼─────────────┤
│    痛点     │    收益     │
└─────────────┴─────────────┘
```

## 二、双钻模型 (Double Diamond)

### 2.1 四阶段结构

```
第一钻石：发现问题
  Discover → Define
  发现阶段 → 定义阶段

第二钻石：解决问题
  Develop → Deliver
  开发阶段 → 交付阶段
```

| 阶段 | 目标 | 核心活动 |
|------|------|----------|
| Discover | 广泛探索 | 用户研究、业务调研 |
| Define | 聚焦问题 | POV陈述、范围界定 |
| Develop | 发散方案 | 原型设计、流程走查 |
| Deliver | 完善方案 | 方案细化、PRD交付 |

## 三、UX五要素

### 3.1 五层模型

```
表现层 → 骨架层 → 结构层 → 范围层 → 战略层
（从具体到抽象）
```

| 层级 | 核心问题 | 设计输出 |
|------|----------|----------|
| 战略层 | 我们要什么？用户要什么？ | 产品目标、用户需求 |
| 范围层 | 做什么？不做什么？ | 功能规格、内容需求 |
| 结构层 | 如何组织？如何交互？ | 信息架构、核心流程 |
| 骨架层 | 如何呈现？如何操作？ | 页面布局、交互方式 |
| 表现层 | 看起来如何？感觉如何？ | 视觉规范、设计原则 |

### 3.2 各层输出模板

**战略层**：
```markdown
## 产品目标
- 商业目标、品牌标识

## 用户需求
- 目标用户、核心需求、使用场景

## 成功指标
- 业务指标、用户指标
```

**范围层**：
```markdown
## 功能规格
| 功能 | 优先级 | 描述 | 依赖 |

## Out of Scope
- 不做什么
```

**结构层**：
```markdown
## 信息架构
树形目录结构

## 核心流程
流程路径
```

**骨架层**：
```markdown
## 页面布局
页面结构说明

## 交互方式
操作反馈、快捷操作
```

## 五、MVP设计方法

### 5.1 功能筛选矩阵

```
业务价值
  高 │ 必须包含 │ 优先包含
─────┼──────────────────
  低 │ 可选包含 │ 暂不包含
     低 ←───────→ 高
        实现难度
```

### 5.2 企业系统上线验证指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 核心流程完成率 | ≥ 90% | 业务方走查核心流程无断点 |
| 异常场景覆盖率 | 100% | 所有定义的异常场景有处理方案 |
| 权限矩阵覆盖率 | 100% | 每个操作有明确的权限定义 |
| 业务规则验证 | 100% | P0 业务规则全部实现 |
| 用户培训覆盖 | ≥ 95% | 目标用户完成操作培训 |
