# 变更影响分析流程

**何时使用**: PRD变更、需求逻辑调整、接口变更、数据库变更时分析影响范围。

---

## 流程步骤

```
1. 识别变更类型
   ├── 新增功能 → 需要新增接口、表、页面
   ├── 逻辑调整 → 现有接口/页面逻辑变化
   ├── 字段变更 → 数据库字段增删改、接口参数变化
   ├── 状态变更 → 状态机调整、枚举值变化
   └── 接口变更 → 参数/返回值变化

2. 分析后端影响
   ├── 接口层 → 哪些接口需要修改/新增
   ├── 服务层 → 哪些服务逻辑需要调整
   ├── 数据层 → 哪些表/字段需要修改/新增
   └── 检查现有代码 → 使用 grep 搜索相关实现

3. 分析前端影响
   ├── 页面层 → 哪些页面需要修改/新增
   ├── 组件层 → 哪些组件需要调整
   ├── 接口调用 → 哪些 API 调用需要修改
   └── 状态管理 → 哪些全局状态需要更新

4. 识别风险点
   ├── 历史数据 → 字段变更是否需要数据迁移
   ├── 兼容性 → 接口变更是否影响已有调用方
   ├── 性能 → 新增功能是否有性能风险
   └── 权限 → 是否涉及权限配置变更

5. 生成影响报告
   └── 使用模板 → @../change-templates.md#影响分析报告
```

---

## 变更类型分析清单

### 新增功能

```
检查项：
□ 需要新增哪些数据库表/字段？
□ 需要新增哪些接口？
□ 需要新增哪些前端页面/路由？
□ 是否需要新增权限配置？
□ 是否需要新增枚举值/字典数据？
```

### 逻辑调整

```
检查项：
□ 影响哪些现有接口？
□ 影响哪些业务规则？
□ 是否影响状态机？
□ 是否影响已有数据？
□ 前端哪些页面需要同步调整？
```

### 字段变更

```
检查项：
□ 数据库字段类型/长度/默认值
□ 接口参数/返回值
□ 前端表单/展示
□ 历史数据处理
□ 是否需要迁移脚本
```

### 状态变更

```
检查项：
□ 状态枚举值变化
□ 状态流转条件变化
□ 现有数据的状态值
□ 前端状态展示/筛选
□ 状态相关业务规则
```

### 接口变更

```
检查项：
□ 请求参数变化
□ 返回值变化
□ 调用该接口的前端页面
□ 是否需要版本兼容
□ 接口文档更新
```

## 分析维度

### 后端影响

| 层级 | 检查内容 |
|------|---------|
| 接口层 | Controller 接口路径、参数、返回值 |
| 服务层 | Service 业务逻辑、校验规则 |
| 数据层 | Entity 字段、Mapper 查询、索引 |
| 枚举/字典 | 状态值、类型值、选项值 |

### 前端影响

| 层级 | 检查内容 |
|------|---------|
| 页面 | 路由、页面组件、布局 |
| 组件 | 表单、表格、弹窗、筛选 |
| API | 接口调用、参数、返回值处理 |
| 状态 | Pinia store、全局状态 |

### 数据影响

| 维度 | 检查内容 |
|------|---------|
| 表结构 | 新增表、新增字段、字段类型变更 |
| 数据迁移 | 默认值、历史数据处理 |
| 索引 | 新增索引、查询性能 |
