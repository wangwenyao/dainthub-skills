# 变更影响分析流程

**何时使用**: PRD变更、需求逻辑调整、接口变更、数据库变更时分析影响范围。

---

## 输入来源

从 product-manager 输出物提取：

| 输入 | 来源 | 提取内容 |
|------|------|---------|
| **功能清单** | PRD 功能架构 | 功能ID → 变更范围识别 |
| **ER图 + 实体属性** | PRD 业务实体模型 | 实体字段 → 数据库影响 |
| **字段约束** | PRD 功能详细描述 | 约束规则 → 接口参数校验 |
| **业务规则** | PRD BR编号 | 规则 → 逻辑影响范围 |
| **状态流转图** | PRD 状态机 | 状态变更 → 前后端联动 |
| **YAML 页面规格** | 线框图 YAML | 前端页面 → 接口调用关系 |

---

## 流程步骤

```
1. 识别变更类型
   ├── 新增功能 → 需要新增接口、表、页面
   ├── 逻辑调整 → 现有接口/页面逻辑变化
   ├── 字段变更 → 数据库字段增删改、接口参数变化
   ├── 状态变更 → 状态机调整、枚举值变化
   └── 接口变更 → 参数/返回值变化

2. 分析后端影响
   ├── 接口层 → 哪些接口需要修改/新增
   ├── 服务层 → 哪些服务逻辑需要调整
   ├── 数据层 → 哪些表/字段需要修改/新增
   └── 检查现有代码 → 使用 grep 搜索相关实现

3. 分析前端影响
   ├── 页面层 → 哪些页面需要修改/新增
   ├── 组件层 → 哪些组件需要调整
   ├── 接口调用 → 哪些 API 调用需要修改
   └── 状态管理 → 哪些全局状态需要更新

4. 识别风险点
   ├── 历史数据 → 字段变更是否需要数据迁移
   ├── 兼容性 → 接口变更是否影响已有调用方
   ├── 性能 → 新增功能是否有性能风险
   └── 权限 → 是否涉及权限配置变更

5. 生成影响报告
   └── 使用模板 → 见下方"影响范围报告模板"
```

---

## 详细分析流程

### 步骤 1: 识别变更类型

**具体操作**:

```
1. 对比新旧 PRD，标记变更点
   - 使用 diff 工具对比 PRD 文档
   - 标记新增、修改、删除的功能/字段/规则

2. 分类变更类型
   - 新增功能: 功能ID 新增、ER图 新增实体
   - 逻辑调整: BR编号 规则变更、状态流转图 变化
   - 字段变更: 实体属性表 字段增删改
   - 状态变更: 状态枚举值 变化、状态流转条件 变化
   - 接口变更: YAML 页面规格 接口调用变化

3. 记录变更清单
   - 每个变更点记录: 变更ID、变更类型、PRD章节、影响范围预估
```

**变更类型判定表**:

| PRD 变化 | 变更类型 | 典型场景 |
|---------|---------|---------|
| 新增功能ID + 新增 ER图实体 | 新增功能 | 新增商品评价功能 |
| BR编号 规则描述变更 | 逻辑调整 | 修改库存扣减规则 |
| 实体属性表 新增/删除字段 | 字段变更 | 商品表新增库存预警字段 |
| 状态枚举值 新增/删除 | 状态变更 | 商品状态新增"预售"状态 |
| YAML 页面规格 接口参数变化 | 接口变更 | 列表页新增筛选条件 |

---

### 步骤 2: 分析后端影响

**具体操作**:

```
1. 定位相关代码文件
   - 使用 grep 搜索实体名、服务名、接口路径
   - 查找 Controller、Service、Mapper、Entity 文件

2. 分析接口层影响
   - 检查 Controller 接口路径、参数、返回值
   - 确定需要新增/修改的接口

3. 分析服务层影响
   - 检查 Service 业务逻辑、校验规则
   - 确定需要修改的业务规则实现

4. 分析数据层影响
   - 检查 Entity 字段定义
   - 检查 Mapper 查询语句
   - 确定需要新增/修改的表、字段、索引

5. 分析枚举/字典影响
   - 检查枚举类定义
   - 检查字典数据配置
   - 确定需要新增/修改的枚举值
```

**代码搜索命令示例**:

```bash
# 搜索实体类
grep -r "class Product" --include="*.java" src/

# 搜索 Controller 接口
grep -r "@RequestMapping.*product" --include="*.java" src/
grep -r "GetMapping.*product" --include="*.java" src/

# 搜索 Service 方法
grep -r "ProductService" --include="*.java" src/
grep -r "createProduct\|updateProduct\|deleteProduct" --include="*.java" src/

# 搜索 Mapper 查询
grep -r "ProductMapper" --include="*.java" src/
grep -r "selectProduct\|insertProduct" --include="*.xml" src/

# 搜索枚举定义
grep -r "ProductStatus" --include="*.java" src/
grep -r "enum.*Status" --include="*.java" src/

# 搜索字段引用
grep -r "stockWarning\|stock_warning" --include="*.java" src/
grep -r "stockWarning\|stock_warning" --include="*.xml" src/

# 搜索前端 API 调用
grep -r "api/product" --include="*.ts" --include="*.vue" src/
grep -r "getProductPage\|createProduct" --include="*.ts" src/
```

### 步骤 3: 分析前端影响

**具体操作**:

```
1. 定位相关页面文件
   - 使用 grep 搜索页面路由、组件名
   - 查找 views、components 文件

2. 分析页面层影响
   - 检查路由配置
   - 检查页面组件结构
   - 确定需要新增/修改的页面

3. 分析组件层影响
   - 检查表单组件字段配置
   - 检查表格组件列配置
   - 检查筛选组件选项配置
   - 确定需要修改的组件

4. 分析接口调用影响
   - 检查 API 定义文件
   - 检查接口调用位置
   - 确定需要修改的接口调用

5. 分析状态管理影响
   - 检查 Pinia store 定义
   - 检查全局状态使用位置
   - 确定需要修改的状态管理
```

**前端代码搜索命令示例**:

```bash
# 搜索页面路由
grep -r "product" --include="*.ts" src/router/

# 搜索页面组件
grep -r "ProductList\|ProductDetail" --include="*.vue" src/views/

# 搜索表单字段配置
grep -r "stockWarning\|stock_warning" --include="*.vue" src/
grep -r "formSchema.*stock" --include="*.ts" src/

# 搜索表格列配置
grep -r "columns.*stock" --include="*.ts" src/
grep -r "VxeGrid.*product" --include="*.vue" src/

# 搜索 API 定义
grep -r "productApi\|/api/product" --include="*.ts" src/api/

# 搜索接口调用
grep -r "getProductPage\|createProduct" --include="*.vue" --include="*.ts" src/

# 搜索状态管理
grep -r "productStore\|useProductStore" --include="*.ts" src/stores/
```

### 步骤 4: 识别风险点

**具体操作**:

```
1. 检查历史数据影响
   - 字段新增: 是否需要默认值填充
   - 字段修改: 是否需要数据迁移
   - 字段删除: 是否需要数据备份
   - 状态变更: 是否需要状态值映射

2. 检查兼容性影响
   - 接口参数新增: 是否影响已有调用方
   - 接口参数删除: 是否需要版本兼容
   - 返回值变化: 是否影响前端展示

3. 检查性能影响
   - 新增查询条件: 是否需要新增索引
   - 新增关联查询: 是否有性能风险
   - 大数据量操作: 是否需要分批处理

4. 检查权限影响
   - 新增功能: 是否需要新增权限配置
   - 功能调整: 是否需要调整权限范围
   - 数据隔离: 是否涉及租户隔离变更
```

---

### 步骤 5: 生成影响报告

**具体操作**:

```
1. 填写变更概述
   - 变更来源、变更时间、变更内容、变更原因

2. 填写影响范围表格
   - 后端影响: 接口层、服务层、数据层、枚举层
   - 前端影响: 页面层、组件层、API层、状态层
   - 数据影响: 表结构、数据迁移、索引

3. 填写风险评估表格
   - 风险项、风险等级、说明、缓解措施

4. 填写同步建议
   - 需要通知的角色、具体任务、优先级

5. 填写时间预估
   - 任务、预估工时、负责方
```

## 变更类型分析清单

### 新增功能

```
检查项：
□ 需要新增哪些数据库表/字段？
□ 需要新增哪些接口？
□ 需要新增哪些前端页面/路由？
□ 是否需要新增权限配置？
□ 是否需要新增枚举值/字典数据？
```

**详细分析步骤**:

| 步骤 | 操作 | 输出 |
|------|------|------|
| 1. 提取 ER图 | 从 PRD 提取新增实体、字段 | 表清单、字段清单 |
| 2. 设计接口 | 根据功能清单设计 CRUD 接口 | 接口清单 |
| 3. 设计页面 | 根据 YAML 页面规格设计页面 | 页面清单、组件清单 |
| 4. 设计权限 | 根据功能ID 设计权限标识 | 权限清单 |
| 5. 设计枚举 | 根据状态枚举设计枚举类 | 枚举清单 |

**影响范围评估方法**:

```
后端影响:
- 新增 Entity 类 → 1 个文件
- 新增 Mapper 接口 + XML → 2 个文件
- 新增 Service 类 → 1 个文件
- 新增 Controller 类 → 1 个文件
- 新增枚举类 → 1 个文件（如有）

前端影响:
- 新增页面组件 → 1-2 个文件
- 新增路由配置 → 修改 1 个文件
- 新增 API 定义 → 1 个文件
- 新增表单/表格组件 → 可能复用现有组件

数据影响:
- 新增表 → 1 个 CREATE TABLE
- 新增索引 → 2-3 个 INDEX
- 新增字典数据 → INSERT 语句
```

**风险等级判定**:

| 条件 | 风险等级 |
|------|---------|
| 新增独立模块，无关联依赖 | 低 |
| 新增模块，需关联现有模块 | 中 |
| 新增核心模块，影响主流程 | 高 |

---

### 逻辑调整

```
检查项：
□ 影响哪些现有接口？
□ 影响哪些业务规则？
□ 是否影响状态机？
□ 是否影响已有数据？
□ 前端哪些页面需要同步调整？
```

**详细分析步骤**:

| 步骤 | 操作 | 输出 |
|------|------|------|
| 1. 定位规则代码 | grep 搜索 BR编号 相关实现 | 受影响文件清单 |
| 2. 分析逻辑变化 | 对比新旧规则，确定变化点 | 逻辑变化清单 |
| 3. 分析接口影响 | 检查接口参数/返回值是否变化 | 接口变化清单 |
| 4. 分析前端影响 | 检查前端调用是否需要调整 | 前端变化清单 |
| 5. 分析数据影响 | 检查历史数据是否需要处理 | 数据处理方案 |

**影响范围评估方法**:

```
后端影响:
- 修改 Service 方法 → 1-2 个文件
- 可能修改 Controller 参数校验 → 1 个文件
- 可能修改 Mapper 查询条件 → 1 个文件

前端影响:
- 修改表单校验规则 → 1 个文件
- 修改表格筛选条件 → 1 个文件
- 修改状态展示逻辑 → 1 个文件

数据影响:
- 可能需要数据迁移 → UPDATE 语句
- 可能需要状态值映射 → UPDATE CASE 语句
```

**风险等级判定**:

| 条件 | 风险等级 |
|------|---------|
| 规则调整，不影响已有数据 | 低 |
| 规则调整，需处理历史数据 | 中 |
| 规则调整，影响核心业务流程 | 高 |

---

```
检查项：
□ 数据库字段类型/长度/默认值
□ 接口参数/返回值
□ 前端表单/展示
□ 历史数据处理
□ 是否需要迁移脚本
```

**详细分析步骤**:

| 步骤 | 操作 | 输出 |
|------|------|------|

| 4. 分析前端影响 | 确定表单/表格变化 | 前端变化清单 |
| 5. 设计数据迁移 | 确定历史数据处理方案 | 数据迁移脚本 |

**影响范围评估方法**:

```
后端影响:
- 修改 Entity 字段 → 1 个文件
- 修改 Mapper XML 字段映射 → 1 个文件
- 可能修改 Service 校验逻辑 → 1 个文件
- 可能修改 Controller 参数校验 → 1 个文件

前端影响:
- 修改表单字段配置 → 1 个文件
- 修改表格列配置 → 1 个文件
- 修改详情展示字段 → 1 个文件
```

**风险等级判定**:

| 条件 | 风险等级 |
|------|---------|
| 新增字段，有默认值，无历史数据依赖 | 低 |

| 修改字段类型/长度，需检查超长数据 | 中 |
| 删除字段，需备份历史数据 | 高 |

## 分析维度

### 后端影响

| 层级 | 检查内容 | 搜索关键词 |
|------|---------|-----------|
| 接口层 | Controller 接口路径、参数、返回值 | `@RequestMapping`, `@GetMapping`, `@PostMapping` |
| 服务层 | Service 业务逻辑、校验规则 | `Service`, `ServiceImpl` |
| 数据层 | Entity 字段、Mapper 查询、索引 | `Entity`, `Mapper`, `select`, `insert` |
| 枚举/字典 | 状态值、类型值、选项值 | `enum`, `Enum`, `Dict` |

### 前端影响

| 层级 | 检查内容 | 搜索关键词 |
|------|---------|-----------|
| 页面 | 路由、页面组件、布局 | `views/`, `router/` |
| 组件 | 表单、表格、弹窗、筛选 | `VbenForm`, `VxeGrid`, `VbenModal` |
| API | 接口调用、参数、返回值处理 | `api/`, `get`, `post` |
| 状态 | Pinia store、全局状态 | `stores/`, `useStore` |

### 数据影响

| 维度 | 检查内容 | 输出物 |
|------|---------|--------|
| 表结构 | 新增表、新增字段、字段类型变更 | CREATE TABLE / ALTER TABLE |
| 数据迁移 | 默认值、历史数据处理 | UPDATE / INSERT |
| 索引 | 新增索引、查询性能 | CREATE INDEX |

## 前后端联动分析方法

### 联动分析流程

```
1. 从接口变更追踪前端影响
   ├── 后端接口参数变化 → 前端 API 调用参数变化
   ├── 后端返回值变化 → 前端数据处理逻辑变化
   └── 后端接口新增 → 前端新增 API 定义

2. 从前端需求追踪后端影响
   ├── 前端新增筛选条件 → 后端新增查询参数
   ├── 前端新增展示字段 → 后端新增返回字段
   └── 前端新增操作按钮 → 后端新增接口

3. 从数据变更追踪前后端影响
   ├── 数据库新增字段 → Entity 新增字段 → 接口返回值新增字段 → 前端展示新增字段
   ├── 数据库字段类型变化 → Entity 字段类型变化 → 接口参数类型变化 → 前端表单字段类型变化
   └── 数据库状态值变化 → 枚举类变化 → 接口返回值变化 → 前端状态展示变化
```

### 联动追踪表

| 变更起点 | 后端影响 | 前端影响 | 联动检查点 |
|---------|---------|---------|-----------|
| 数据库新增字段 | Entity + Mapper + Service + Controller | API + 表单 + 表格 + 详情 | 接口返回值是否包含新字段 |
| 数据库修改字段类型 | Entity + Mapper | API + 表单校验 | 接口参数类型是否同步 |
| 数据库新增状态值 | 枚举类 + Service | API + 状态筛选 + 状态展示 | 前端是否展示新状态标签 |
| 接口新增参数 | Controller + Service | API + 表单 + 筛选 | 前端是否传递新参数 |
| 接口新增返回字段 | Controller + Service | API + 表格 + 详情 | 前端是否展示新字段 |
| 前端新增筛选条件 | Controller + Mapper + 索引 | API + 筛选组件 | 后端是否支持新筛选条件 |

### 联动验证清单

```
□ 数据库字段变更 → Entity 是否同步
□ Entity 字段变更 → Mapper XML 是否同步
□ Entity 字段变更 → 接口返回值是否包含
□ 接口参数变更 → 前端 API 调用是否同步
□ 接口返回值变更 → 前端数据处理是否同步
□ 状态枚举变更 → 前端状态展示是否同步
□ 前端筛选条件新增 → 后端是否支持查询
□ 前端操作按钮新增 → 后端是否提供接口
```

## 风险等级定义

### 风险等级判定标准

| 风险等级 | 定义 | 典型场景 |
|---------|------|---------|
| **高** | 影响核心业务流程、需数据迁移、有兼容性风险 | 删除字段、修改状态值含义、修改接口参数类型 |
| **中** | 影响多个模块、需历史数据处理、需前后端同步 | 新增必填字段、新增状态值并调整流转、新增查询条件需索引 |
| **低** | 影响单一模块、无历史数据依赖、向后兼容 | 新增可选字段、新增独立功能、新增可选参数 |

### 高风险场景示例

| 场景 | 风险点 | 缓解措施 |
|------|--------|---------|
| 删除数据库字段 | 历史数据丢失、代码引用报错 | 先备份数据、先删除代码引用、再删除字段 |
| 修改状态值含义 | 历史数据状态值含义变化 | 设计状态值映射脚本、更新历史数据 |
| 修改接口参数类型 | 前端调用报错、兼容性问题 | 设计版本兼容方案、分阶段迁移 |
| 修改核心业务规则 | 影响主流程、可能产生业务错误 | 充分测试、灰度发布、监控告警 |

### 中风险场景示例

| 场景 | 风险点 | 缓解措施 |
|------|--------|---------|
| 新增必填字段 | 历史数据无值、前端需同步修改 | 设计默认值填充方案、前后端同步开发 |
| 新增状态值并调整流转 | 历史数据状态流转条件变化 | 设计状态值映射、更新历史数据状态 |
| 新增查询条件需索引 | 查询性能下降 | 先新增索引、再上线功能 |
| 修改字段长度 | 历史数据可能超长 | 先检查超长数据、再修改字段 |

### 低风险场景示例

| 场景 | 风险点 | 缓解措施 |
|------|--------|---------|
| 新增可选字段 | 无历史数据依赖 | 设置默认值、无需数据迁移 |
| 新增独立功能 | 无关联依赖 | 独立开发、独立测试 |
| 新增可选参数 | 向后兼容 | 前端可选传递、后端兼容处理 |
