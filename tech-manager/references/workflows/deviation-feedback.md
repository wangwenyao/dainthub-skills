# 实现偏差反馈流程

**何时使用**: 开发过程中或实现验证后，发现代码实现与 PRD 存在偏差，需要反馈给产品经理确认。

---

## 触发条件

以下任一情况出现时，启动偏差反馈：

| 触发场景 | 示例 |
|---------|------|
| 技术限制导致方案调整 | 某字段不支持实时计算，改为异步批处理 |
| 发现 PRD 逻辑缺陷 | 审批流转存在死锁、状态机不完整 |
| 接口与 PRD 字段不完全对应 | 合并/拆分字段、类型调整 |
| 实现验证发现功能偏差 | 验证报告中存在"不通过"项 |
| 新增技术约束 | 性能限制、安全约束、第三方依赖限制 |
| 开发过程中主动优化 | 简化流程、合并步骤、增加自动化 |

---

## 流程步骤

```
1. 识别偏差
   ├── 开发过程中发现 → 开发者向技术经理上报
   └── 实现验证中发现 → 验证报告中的"不通过"项

2. 偏差分类
   ├── 技术限制型 → 技术上无法按PRD实现，需调整方案
   ├── 逻辑缺陷型 → PRD 存在逻辑矛盾或遗漏
   ├── 优化建议型 → 技术侧发现更优实现方式
   └── 约束新增型 → 实现中发现新的限制条件

3. 评估影响
   ├── 对用户体验的影响（高/中/低/无）
   ├── 对功能完整性的影响
   ├── 对后续迭代的影响
   └── 是否有替代方案

4. 生成偏差反馈报告
   └── 使用模板 → @../change-templates.md#实现偏差反馈

5. 提交给产品经理确认
   ├── 偏差内容 + 影响评估 + 建议方案
   └── 等待产品经理决策

6. 记录决策结果
   ├── 接受偏差 → 更新PRD对应章节
   ├── 坚持PRD → 开发者按PRD重新实现
   └── 协商调整 → 产品+技术共同确定新方案
```

---

## 偏差分类详解

### 技术限制型

```
场景：PRD要求实时计算库存，但数据量大无法在200ms内完成
偏差：改为定时任务每5分钟同步，页面展示"约X件"
影响：用户看到的库存可能有5分钟延迟
建议：接受异步方案，页面标注"库存仅供参考"
```

### 逻辑缺陷型

```
场景：PRD定义的审批状态机缺少"撤回"到"草稿"的流转
偏差：开发时发现用户提交后无法撤回修改
影响：用户体验缺失
建议：补充撤回流程（已提交→撤回→草稿）
```

### 优化建议型

```
场景：PRD要求手动选择收货地址，但可以自动定位
偏差：开发时增加了自动定位+手动修改的组合方式
影响：用户体验提升，无负面影响
建议：接受优化方案
```

### 约束新增型

```
场景：使用的第三方支付SDK不支持PRD要求的"组合支付"
偏差：仅支持单一支付方式
影响：MVP阶段组合支付不可用
建议：MVP先上线单一支付，后续迭代支持组合支付
```

## 偏差严重等级

| 等级 | 定义 | 处理要求 |
|------|------|---------|
| **P0-阻塞** | 核心功能无法按PRD实现 | 必须产品经理确认后才能继续 |
| **P1-重要** | 非核心功能偏差，影响用户体验 | 提交反馈，等待确认 |
| **P2-一般** | 细节偏差，不影响核心流程 | 提交反馈，可先按技术方案实现 |
| **P3-建议** | 优化建议，提升体验 | 提交反馈，记录即可 |

### 处理时效

| 等级 | 产品经理响应时效 | 开发处理 |
|------|----------------|---------|
| P0 | 立即确认 | 阻塞等待 |
| P1 | 1个工作日内 | 可先做其他任务 |
| P2 | 3个工作日内 | 按技术方案先行 |
| P3 | 下次迭代评审 | 按技术方案先行 |

## 实现验证串联

实现验证完成后，验证报告中的"不通过"项自动进入偏差反馈流程：

```
实现验证报告
    ├── ✅ 通过项 → 无需反馈
    └── ❌ 不通过项
         ├── 分析原因
         │   ├── 开发遗漏 → 开发者补充实现（不需要反馈产品）
         │   ├── 技术限制 → 生成偏差反馈 → 产品经理确认
         │   ├── PRD逻辑问题 → 生成偏差反馈 → 产品经理确认
         │   └── 需求理解偏差 → 与产品经理对齐理解
         └── 输出
             ├── 开发补充任务（遗漏类）
             └── 偏差反馈报告（限制/缺陷类）
```

## 决策记录

产品经理确认后，记录决策结果：

```markdown
## 偏差决策记录

| # | 偏差描述 | 等级 | 产品决策 | PRD更新 | 决策时间 |
|---|---------|------|---------|---------|---------|
| 1 | 库存改为异步计算 | P1 | 接受 | 更新3.2节 | 2026-04-03 |
| 2 | 审批增加撤回流程 | P0 | 接受 | 补充4.1节 | 2026-04-03 |
| 3 | 组合支付延后 | P1 | 接受 | 移至v2.0 | 2026-04-03 |

决策后行动：
- [ ] PRD已更新对应章节
- [ ] 开发者已知悉最终方案
- [ ] 测试用例已同步调整
```
