# 实现验证流程

**何时使用**: 开发完成后对照 PRD 验证代码实现是否满足需求规格。

---

## 流程步骤

```
1. 获取验证输入
   ├── PRD文档（功能需求、业务规则、验收标准）
   ├── 接口文档（后端已实现的接口）
   └── 代码实现（前端页面、后端服务）

2. 功能完整性验证
   ├── PRD定义的功能是否都有对应接口
   ├── 前端页面是否覆盖所有功能入口
   └── 检查是否有遗漏功能

3. 接口一致性验证
   ├── 前端调用的接口是否在后端存在
   ├── 接口参数是否与定义一致
   └── 返回值是否被正确使用

4. 字段完整性验证
   ├── PRD定义的字段是否在接口中体现
   ├── 数据库字段是否与接口对应
   └── 前端表单字段是否与接口参数对应

5. 业务规则验证
   ├── 校验规则是否实现
   ├── 状态流转是否正确
   └── 权限控制是否到位

6. 输出验证报告
   └── 列出符合项、缺失项、风险项
```

---

## 验证维度

### 功能完整性

```
验证方法：
1. 提取 PRD 功能清单
2. 检查后端接口覆盖情况
3. 检查前端页面覆盖情况

检查项：
□ 每个 P0 功能都有对应接口
□ 每个功能都有前端入口
□ CRUD 操作是否完整
□ 搜索/筛选/排序/分页是否实现
```

### 接口一致性

```
验证方法：
1. 提取前端 API 调用列表
2. 对比后端接口定义
3. 检查参数和返回值

检查项：
□ 前端调用的接口路径在后端存在
□ 请求参数名称、类型一致
□ 返回值字段名称、类型一致
□ 错误码处理一致
```

### 字段完整性

```
验证方法：
1. 提取 PRD 字段定义
2. 检查数据库表结构
3. 检查接口参数/返回值
4. 检查前端表单字段

检查项：
□ PRD 定义的字段在数据库中存在
□ 数据库字段在接口中有体现
□ 接口字段在前端有使用
□ 字段类型、长度、必填一致
```

### 业务规则

```
验证方法：
1. 提取 PRD 业务规则（BR编号）
2. 检查后端代码实现
3. 检查前端校验

检查项：
□ 字段校验规则（长度、格式、范围）
□ 状态流转条件
□ 权限控制逻辑
□ 异常处理逻辑
```
