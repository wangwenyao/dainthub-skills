# ch6 — 项目实施计划

受众：业务决策层 + 项目管理。里程碑和风险用业务语言描述。

## 结构与写作规则

### 6.1 整体交付计划

阶段划分原则：
- 第一阶段（MVP）：P0 功能 + 系统管理基础能力，跑通核心业务流程
- 第二阶段：P1 功能，完整业务功能覆盖，通过 UAT
- 第三阶段（可选）：P2 功能，统计分析和优化类功能

工期估算参考：

| 功能规模 | 基础工期 |
|---|---|
| <5 个模块，无复杂模块 | 1-2 个月 |
| 5-15 个模块，1-3 个复杂模块 | 3-5 个月 |
| >15 个模块，3 个以上复杂模块 | 6 个月以上 |

复杂模块（状态机、数据权限、外部集成）工期比标准模块增加 50%-100%。

里程碑表格：

| 阶段 | 时间周期 | 主要交付内容 | 验收标准 |
|---|---|---|---|
| 第一阶段（MVP）| 第 1-N 周 | {P0 功能清单} | 核心业务流程可完整跑通 |
| 第二阶段 | 第 N+1-M 周 | {P1 功能清单} | 全部一期功能完成，通过 UAT |
| 上线准备 | 第 M+1-M+2 周 | 性能测试、数据迁移、用户培训 | 通过上线检查清单 |
| 正式上线 | 第 M+3 周 | 生产环境部署，切换上线 | 系统正常运行 72 小时 |

### 6.2 团队组织与职责

| 角色 | 主要职责 |
|---|---|
| 项目经理 | 进度管理、需求协调、风险跟踪、客户沟通 |
| 技术架构师 | 技术方案设计、关键技术决策、代码评审 |
| 后端开发 | Spring Boot 业务开发、接口开发 |
| 前端开发 | Vue3 页面开发、前后端联调 |
| 测试工程师 | 功能测试、接口测试、性能测试 |
| 运维工程师 | 环境搭建、部署、监控配置 |

客户侧配合（需客户提供）：业务负责人（需求确认、UAT 组织）、IT 管理员（服务器资源和网络）、关键业务用户（UAT 参与）。

### 6.3 风险识别与应对

| 风险 | 可能性 | 业务影响 | 应对措施 |
|---|---|---|---|
| 需求频繁变更 | 中 | 延期交付 | 每阶段开始前锁定需求，变更走变更管理流程 |
| 外部系统接口不稳定 | 中（如有集成）| 集成模块延期 | 提前获取接口文档，开发阶段用 Mock，联调预留缓冲 |
| 客户 UAT 时间不足 | 中 | 上线推迟 | 提前 2 周通知客户准备，明确验收时间节点 |
| 服务器资源准备延迟 | 低 | 测试和上线推迟 | 提前 4 周确认资源清单，客户 IT 配合 |

### 6.4 质量保障机制
- **代码评审**：所有代码合并前须经过同行评审，重点关注业务逻辑正确性和域边界合规性
- **测试覆盖**：所有 P0、P1 功能的正常路径和主要异常路径均需测试覆盖
- **里程碑评审**：每阶段结束组织评审会，客户业务方和技术方共同确认后进入下一阶段
- **上线检查清单**：功能验证、性能指标达标、安全扫描、备份验证、回滚方案确认
