# implicit-inference

## 规则

对每类显性功能，按下表系统性地补全隐性需求。补全后的需求在文档中显式呈现，让客户确认，不能默默假设。

### 数据管理类的隐性需求

| 隐性需求 | 具体内容 |
|---|---|
| 数据完整性 | 必填校验（前后端双重）、唯一性约束、关联数据级联处理规则 |
| 操作历史 | 创建人、创建时间、修改人、修改时间（四个基础审计字段）；关键字段变更记录 |
| 附件支持 | 文件/图片上传、查看、下载、删除权限控制 |
| 批量操作 | Excel 导入（模板下载 + 上传解析 + 错误反馈）、批量导出 |
| 软删除 | 停用/归档状态替代物理删除 |
| 列表筛选 | 多条件组合筛选、分页 |

### 流程审批类的隐性需求

| 隐性需求 | 具体内容 |
|---|---|
| 消息通知 | 待审通知、结果通知、超时催办（站内信必须；邮件可选）|
| 审批记录 | 每节点的操作人、时间、意见完整记录；审批轨迹时间线可查 |
| 流程状态可视化 | 申请人可看当前进度（当前节点、剩余步骤）|
| 特殊操作 | 撤回（审批未开始前）、驳回后重新提交 |
| 待办中心 | 统一待办入口，待办/已办/我发起 分类 |
| **关键判断（必须在方案中明确）** | 审批流程是固定的还是可配置的？两者复杂度差异极大，不能含糊 |

### 统计报表类的隐性需求

| 隐性需求 | 具体内容 |
|---|---|
| 数据权限隔离 | 不同角色/组织只看自己权限范围内的数据（最容易遗漏、上线后最容易出问题）|
| 时间维度筛选 | 按日/周/月/季度/年筛选；支持自定义时间范围 |
| 导出 | Excel 导出；数据量大时需异步导出（后台生成，完成通知下载）|
| 刷新策略 | 实时查询 vs 定时预聚合（数据量决定选哪种，方案中需说明）|

### 权限控制类的隐性需求

| 隐性需求 | 具体内容 |
|---|---|
| 账号安全 | 密码复杂度、登录失败锁定、锁定解除方式 |
| 登录日志 | 时间、IP、设备、成功/失败状态记录 |
| 会话管理 | Token 超时策略；同账号多处登录的处理策略（允许/踢旧）|
| 组织关联 | 用户归属部门；组织变化时的用户归属处理 |

### 系统集成类的隐性需求

| 隐性需求 | 具体内容 |
|---|---|
| 容错与重试 | 外部不可用时的降级策略；失败重试次数和间隔；最终失败告警 |
| 数据一致性 | 以谁的数据为准（主从关系）；对账机制 |
| 接口鉴权 | 外部调用本系统的鉴权方式；本系统调用外部的凭证管理（不硬编码）|

### 移动端适配场景的隐性需求（如客户提及移动端）

| 隐性需求 | 具体内容 |
|---|---|
| 响应式布局 | 关键页面适配移动端屏幕；操作按钮位置调整 |
| 触屏交互 | 按钮尺寸适配触屏；手势操作（如下拉刷新）|
| 离线模式 | 关键数据本地缓存；离线后数据同步策略（可选）|
| 推送通知 | 移动端消息推送集成（可选）|

### 多租户场景的隐性需求（如客户提及多组织/多公司）

| 隐性需求 | 具体内容 |
|---|---|
| 租户隔离 | 数据按租户隔离（租户 ID 字段）；跨租户查询权限控制 |
| 数据归属 | 创建数据自动关联当前租户；数据导入时租户归属规则 |
| 租户管理 | 租户创建/停用/删除；租户级配置（如 logo、主题色）|
| 计费统计 | 按租户统计使用量；租户级资源限制（可选）|

### 多语言支持场景的隐性需求（如客户提及国际化/海外部署）

| 隐性需求 | 具体内容 |
|---|---|
| 国际化字段 | 业务表增加多语言字段（如 `name_en`、`name_zh`）|
| 语言切换 | 用户级语言偏好设置；界面动态切换语言 |
| 翻译管理 | 系统文本翻译维护；业务数据翻译导入 |
| 区域适配 | 时区处理；日期格式本地化；数字格式本地化 |

### 所有类型共有的隐性需求

无论客户描述了什么功能，以下需求在每个系统中都必然存在：

- 用户认证（登录/登出/Token 刷新）
- 基于角色的访问控制
- 操作审计日志（关键操作的操作人和时间）
- 统一错误提示（友好的错误信息，非技术报错）
- 表单防重复提交
