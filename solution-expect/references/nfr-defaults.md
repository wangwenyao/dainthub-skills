# nfr-defaults

## 规则

客户未明确非功能需求时，按以下默认值填充。所有使用了默认值的地方，在文档中标注：
> *以下指标为基于项目规模的默认假设，请客户确认或调整*

### 按项目规模的基准默认值

| 指标 | 小型（<3人月） | 中型（3-12人月） | 大型（>1年）|
|---|---|---|---|
| 并发用户数 | 20-50 人 | 100-500 人 | 1000+ 人 |
| 峰值 QPS | 50 | 200 | 1000+ |
| 接口响应时间 | P95 < 3 秒 | P95 < 2 秒 | P95 < 1 秒 |
| 系统可用性 | 99%（年停机<88h）| 99.5%（年停机<44h）| 99.9%（年停机<9h）|
| 数据量级 | 单表<100万行 | 单表<1000万行 | 单表可能超亿行 |
| 部署方式 | 单机 | 双实例+负载均衡 | 集群 |

### 按行业的叠加要求

| 行业 | 叠加要求 |
|---|---|
| 金融银行 | 等保三级、数据不出境、操作日志留存≥6月、敏感字段加密存储、密码策略严格 |
| 政务政府 | 等保二级起步、确认国产化替代要求、数据分级分类、政务内网部署 |
| 医疗健康 | 患者数据严格保护、数据留存期限法规要求、所有访问审计记录 |
| 通用企业 | HTTPS、SQL注入/XSS防护、JWT鉴权、登录失败锁定 |

### 使用优先级

1. 客户明确提出的指标 → 直接使用，标注"客户需求"
2. 可从业务规模推算的指标 → 推算后使用，说明推算依据
3. 无法推算的 → 使用上表默认值，标注"默认假设"
