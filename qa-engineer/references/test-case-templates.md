## 一、测试用例标准格式

### 1.1 基础格式

```markdown
### TC-{编号}: {场景描述}

| 属性 | 内容 |
|------|------|
| 来源 | PRD AC-{编号} / BR-{编号} |
| 优先级 | P0 / P1 / P2 |
| 测试类型 | 正常路径 / 边界值 / 异常路径 / 状态迁移 |
| 前置条件 | {条件列表} |
| 测试数据 | {数据准备说明} |

**测试步骤**:
1. {步骤1}
2. {步骤2}
...

**预期结果**:
- {结果点1}
- {结果点2}

**关联缺陷**: DEF-{编号}（如有）
```

### 1.2 Given/When/Then 格式

```markdown
### TC-{编号}: {场景描述}

**Given (前置条件)**:
- {条件1}
- {条件2}

**When (执行操作)**:
1. {操作1}
2. {操作2}

**Then (预期结果)**:
- {结果1}
- {结果2}

**测试数据**:
- {数据说明}
```

## 二、边界值测试模板

### 2.1 数值边界

```markdown
### TC-B{编号}: {字段}_数值边界测试

**测试对象**: {字段名}
**边界定义**: 最小值={min}, 最大值={max}

| 测试点 | 输入值 | 预期结果 | 用例编号 |
|-------|-------|---------|---------|
| 下界外 | {min-1} | 失败/拒绝 | TC-B{n}1 |
| 下界点 | {min} | 成功 | TC-B{n}2 |
| 上界点 | {max} | 成功 | TC-B{n}3 |
| 上界外 | {max+1} | 失败/拒绝 | TC-B{n}4 |
| 正常值 | {正常值} | 成功 | TC-B{n}5 |

**示例: 订单金额边界**

| 测试点 | 输入值 | 预期结果 | 用例编号 |
|-------|-------|---------|---------|
| 下界外 | 0 | 失败（金额必须大于0） | TC-B01 |
| 下界点 | 1 | 成功 | TC-B02 |
| 上界点 | 100000 | 成功 | TC-B03 |
| 上界外 | 100001 | 失败（超过最大金额） | TC-B04 |
| 正常值 | 500 | 成功 | TC-B05 |
```

### 2.2 字符串长度边界

```markdown
### TC-B{编号}: {字段}_长度边界测试

**测试对象**: {字段名}
**边界定义**: 最小长度={minLen}, 最大长度={maxLen}

| 测试点 | 输入值 | 预期结果 | 用例编号 |
|-------|-------|---------|---------|
| 空值 | "" | 失败（必填） | TC-B{n}1 |
| 下界点 | 1字符 | 成功 | TC-B{n}2 |
| 上界点 | {maxLen}字符 | 成功 | TC-B{n}3 |
| 上界外 | {maxLen+1}字符 | 失败/截断 | TC-B{n}4 |

**示例: 用户名长度边界**

| 测试点 | 输入值 | 预期结果 | 用例编号 |
|-------|-------|---------|---------|
| 空值 | "" | 失败（用户名必填） | TC-B11 |
| 下界点 | "A" | 成功 | TC-B12 |
| 上界点 | "Username20Chars" | 成功 | TC-B13 |
| 上界外 | "Username21Characters" | 失败（超长） | TC-B14 |
```

### 2.3 数量边界

```markdown
### TC-B{编号}: {对象}_数量边界测试

**测试对象**: {对象名}
**边界定义**: 最小数量={min}, 最大数量={max}

| 测试点 | 数量 | 预期结果 | 用例编号 |

**示例: 购物车商品数量边界**

| 测试点 | 数量 | 预期结果 | 用例编号 |
```

## 三、状态迁移测试模板

### 3.1 状态迁移矩阵

```markdown
### TC-S{编号}: {对象}_状态迁移测试

**状态列表**: {状态1}, {状态2}, {状态3}, ...

**迁移规则**:
| 当前状态 | 操作 | 目标状态 | 允许 | 用例编号 |
|---------|------|---------|------|---------|
| {状态1} | {操作1} | {状态2} | ✅ | TC-S{n}1 |
| {状态1} | {操作2} | {状态3} | ✅ | TC-S{n}2 |
| {状态1} | {非法操作} | {状态4} | ❌ | TC-S{n}3 |

**示例: 订单状态迁移**

**状态列表**: 待支付, 已支付, 已发货, 已完成, 已取消

| 当前状态 | 操作 | 目标状态 | 允许 | 用例编号 |
|---------|------|---------|------|---------|
| 待支付 | 支付成功 | 已支付 | ✅ | TC-S01 |
| 待支付 | 用户取消 | 已取消 | ✅ | TC-S02 |
| 待支付 | 超时未支付 | 已取消 | ✅ | TC-S03 |
| 待支付 | 商家发货 | 已发货 | ❌ | TC-S04 |
| 已支付 | 商家发货 | 已发货 | ✅ | TC-S05 |
| 已支付 | 用户取消 | 已取消 | ❌ | TC-S06 |
| 已发货 | 用户确认收货 | 已完成 | ✅ | TC-S07 |
| 已发货 | 用户取消 | 已取消 | ❌ | TC-S08 |
```

### 3.2 状态路径测试

```markdown
### TC-S{编号}: {对象}_完整路径测试

**正常路径**: {状态1} → {状态2} → {状态3} → {最终状态}

**测试步骤**:
1. 初始状态: {状态1}
2. 执行: {操作1}
3. 验证: 状态变为 {状态2}
4. 执行: {操作2}
5. 验证: 状态变为 {状态3}
...

**示例: 订单正常流程**

**路径**: 待支付 → 已支付 → 已发货 → 已完成

**测试步骤**:
1. 创建订单 → 状态=待支付
2. 执行支付 → 状态=已支付
3. 商家发货 → 状态=已发货
4. 用户确认 → 状态=已完成
```

## 四、正交实验测试模板

### 4.1 正交表设计

```markdown
### TC-O{编号}: {功能}_多参数组合测试

**测试因子**:
| 因子 | 名称 | 水平1 | 水平2 | 水平3 |
|------|------|-------|-------|-------|
| A | {因子A名} | {值1} | {值2} | {值3} |
| B | {因子B名} | {值1} | {值2} | - |
| C | {因子C名} | {值1} | {值2} | - |

**正交表**: L{行数}({水平数}^{因子数})

**测试组合**:
| 用例编号 | A | B | C | 预期结果 |
|---------|---|---|---|---------|
| TC-O{n}1 | 水平1 | 水平1 | 水平1 | {结果} |
| TC-O{n}2 | 水平1 | 水平2 | 水平2 | {结果} |
| TC-O{n}3 | 水平2 | 水平1 | 水平2 | {结果} |
| ... | ... | ... | ... | ... |

**示例: 商品搜索组合测试**

**因子**:
| 因子 | 名称 | 水平1 | 平2 | 水平3 |
|------|------|-------|-------|-------|
| A | 关键词类型 | 商品名 | 品牌 | SKU |
| B | 价格范围 | 无 | 有区间 | - |
| C | 分类筛选 | 无 | 一级 | 二级 |
| D | 排序方式 | 默认 | 价格升 | 价格降 |

**正交表**: L9(3^4)

| 用例编号 | A | B | C | D | 预期结果 |
|---------|---|---|---|---|---------|
| TC-O01 | 商品名 | 无 | 无 | 默认 | 返回结果 |
| TC-O02 | 商品名 | 有 | 一级 | 价格升 | 返回筛选结果 |
| TC-O03 | 品牌 | 无 | 一级 | 价格降 | 返回筛选结果 |
| TC-O04 | 品牌 | 有 | 无 | 默认 | 返回结果 |
| TC-O05 | SKU | 无 | 二级 | 价格升 | 返回筛选结果 |
| TC-O06 | SKU | 有 | 一级 | 价格降 | 返回筛选结果 |
| TC-O07 | 商品名 | 有 | 二级 | 默认 | 返回筛选结果 |
| TC-O08 | 品牌 | 有 | 二级 | 价格升 | 返回筛选结果 |
| TC-O09 | SKU | 无 | 无 | 价格降 | 返回结果 |
```

## 六、异常路径测试模板

### 6.1 异常场景模板

```markdown
### TC-X{编号}: {功能}_异常路径测试

**异常类型**: {异常类型描述}

**测试步骤**:
1. {触发异常的操作}
2. {系统响应}

**预期结果**:
- {错误提示}
- {系统行为}

**示例: 网络中断**

**异常类型**: 网络连接中断

**测试步骤**:
1. 用户点击提交按钮
2. 断开网络连接
3. 观察系统响应

**预期结果**:
- 显示"网络连接失败，请稍后重试"
- 数据未被提交
- 用户可以重新尝试
```

### 6.2 常见异常场景清单

| 异常类型 | 场景 | 用例编号 |
|---------|------|---------|
| 数据异常 | 空值、无效值、超范围 | TC-X{n}1~10 |
| 网络异常 | 断网、超时、慢速 | TC-X{n}11~15 |
| 权限异常 | 无权限、未登录 | TC-X{n}16~20 |
| 并发异常 | 重复提交、同时操作 | TC-X{n}21~25 |
| 系统异常 | 服务异常、数据库异常 | TC-X{n}26~30 |

## 八、测试用例编号规则

```
TC-{模块编号}{功能编号}{序列号}

模块编号:
├── TC-01xx: 用户管理
├── TC-02xx: 商品管理
├── TC-03xx: 订单管理
├── TC-04xx: 支付管理
├── TC-05xx: 认证授权
└── ...

功能编号:
├── TC-0101xx: 登录功能
├── TC-0102xx: 注册功能
├── TC-0103xx: 用户信息管理
└── ...

序列号细分:
├── TC-{n}01~09: 正常路径
├── TC-{n}10~19: 边界值测试
├── TC-{n}20~29: 异常路径
├── TC-{n}30~39: 状态迁移
├── TC-{n}40~49: 等价类测试
├── TC-{n}50~59: 正交实验
└── TC-{n}E0x: 特殊场景
```

## 九、测试用例质量检查清单

```markdown
□ 编号规范：符合编号规则
□ 来源标注：标注PRD AC/BR编号
□ 优先级明确：P0/P1/P2标注
□ 类型标注：正常/边界/异常/状态
□ 前置条件完整：包含所有必要条件
□ 测试步骤清晰：步骤编号、操作明确
□ 预期结果具体：可验证、可判断
□ 测试数据准备：数据需求说明
□ 关联缺陷：如有缺陷，标注编号
□ 无冗余：不重复覆盖已测试场景
```
