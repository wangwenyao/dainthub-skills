# 测试报告生成流程

**何时使用**: 测试执行完成后，汇总测试结果，生成测试报告。

---

## 流程步骤

```
1. 收集测试数据
   ├── 测试执行记录
   ├── 缺陷记录
   ├── 覆盖率报告
   └── 测试计划文档

2. 统计测试结果
   ├── 用例执行统计
   ├── 通过率统计
   ├── 缺陷统计
   └── 覆盖率统计

3. 分析质量状态
   ├── 评估测试覆盖率
   ├── 分析缺陷分布
   ├── 识别风险项
   └── 评估上线可行性

4. 编写测试报告
   ├── 测试执行摘要
   ├── 详细统计数据
   ├── 问题清单
   ├── 风险提示
   └── 结论与建议

5. 输出测试报告
   └── test-report-{模块名}-v{版本}.md
```

---

## 与 tech-manager 协作

| 报告输出 | tech-manager 使用 |
|---------|------------------|
| 测试通过率 | 实现验证参考 |
| 缺陷统计 | 偏差分析输入 |
| 风险提示 | 上线决策参考 |
| 后续行动 | 任务分配依据 |

### 上线决策支持

```
tech-manager 收到测试报告后:

1. 检查阻塞项
   ├── 致命缺陷 → 阻塞上线
   └── 严重缺陷 → 评估是否阻塞

2. 评估质量指标
   ├── 通过率 ≥ 95% → 通过
   └── 覆盖率达标 → 通过

3. 决策上线/延后
   ├── 全通过 → 安排上线
   └── 存在风险 → 通知 product-manager 决策
```
