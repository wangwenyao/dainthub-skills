# 缺陷管理流程

**何时使用**: 缺陷生命周期管理，从发现到关闭的完整流程。

---

## 缺陷生命周期

```
    ┌─────────┐
    │  新建   │ ← 测试发现缺陷，创建缺陷记录
    └────┬────┘
         ↓
    ┌─────────┐
    │  分配   │ ← 分配给对应开发
    └────┬────┘
         ↓
    ┌─────────┐
    │  修复   │ ← 开发修复完成，标记为已修复
    └────┬────┘
         ↓
    ┌─────────┐
    │  验证   │ ← 测试验证修复结果
    └────┬────┘
         │
    ┌────┴────┐
    ↓         ↓
┌───────┐ ┌───────┐
│ 关闭  │ │ 重开  │ ← 验证不通过，重新打开
└───────┘ └───────┘
         │
         ↓
    ┌─────────┐
    │  重分配 │ ← 重新分配给开发
    └─────────┘
```

---

## 流程步骤

```
1. 缺陷发现与记录
   ├── 测试执行中发现问题
   ├── 创建缺陷记录
   ├── 记录缺陷详情（描述、截图、日志）
   └── 标记状态为「新建」

2. 缺陷分配
   ├── 分析缺陷所属模块
   ├── 分配给对应开发
   └── 标记状态为「分配」

3. 缺陷修复
   ├── 开发分析缺陷原因
   ├── 修复代码
   ├── 单元测试验证修复
   └── 标记状态为「已修复」

4. 缺陷验证
   ├── 测试执行验证
   ├── 确认修复有效 → 标记「关闭」
   ├── 验证不通过 → 标记「重开」
   └── 附加验证结果说明

5. 缺陷关闭
   ├── 确认所有验证通过
   ├── 记录关闭时间
   └── 更新缺陷日志
```

## 严重程度定义

| 等级 | 定义 | 示例 | 处理要求 |
|------|------|------|---------|
| **致命** | 系统崩溃、数据丢失、安全漏洞 | 数据库崩溃、支付失败 | 立即修复，阻塞上线 |
| **严重** | 核心功能无法使用 | 登录失败、订单无法创建 | 优先修复，当日完成 |
| **一般** | 功能异常但有替代方案 | 搜索结果不准确 | 正常修复，版本内完成 |
| **轻微** | UI问题、文案错误 | 按钮颜色不对、提示语不准确 | 低优先级，有空时修复 |

---

## 优先级定义

| 优先级 | 定义 | 与严重程度关系 | 处理时限 |
|-------|------|--------------|---------|
| **P0** | 最高优先级 | 致命缺陷 → P0 | 立即处理 |
| **P1** | 高优先级 | 严重缺陷 → P1 | 当日处理 |
| **P2** | 中优先级 | 一般缺陷 → P2 | 3日内处理 |
| **P3** | 低优先级 | 轻微缺陷 → P3 | 版本内处理 |

## 缺陷报告流程

### 发现缺陷时

```
1. 立即记录缺陷详情
2. 拍摄截图/录屏
3. 收集日志信息
4. 尝试复现确认
5. 创建缺陷单

输出 → defect-log 更新
```

### 开发修复后

```
1. 开发提交修复代码
2. 运行单元测试验证
3. 标记状态为「已修复」
4. 通知测试验证

等待 → 测试验证
```

### 测试验证后

```
通过:
├── 标记状态为「关闭」
├── 记录关闭时间
└── 更新缺陷日志

不通过:
├── 标记状态为「重开」
├── 记录验证失败原因
├── 重新分配
└── 附加新的复现信息
```

## 缺陷度量指标

| 指标 | 计算公式 | 目标值 |
|------|---------|--------|
| 缺陷密度 | 缺陷数 / 功能数 | < 0.5 |
| 缺陷修复率 | 已关闭 / 总缺陷 | ≥ 90% |
| 缺陷重开率 | 重开数 / 总缺陷 | < 5% |
| 平均修复时长 | Σ修复时长 / 修复数 | < 2天 |
| 平均验证时长 | Σ验证时长 / 验证数 | < 0.5天 |

## 缺陷预防

### 常见缺陷类型及预防

| 缺陷类型 | 原因 | 预防措施 |
|---------|------|---------|
| 数据校验缺失 | 边界值未考虑 | 边界值测试覆盖 |
| 状态流转错误 | 状态机理解偏差 | 状态迁移测试 |
| 并发问题 | 未考虑并发场景 | 并发测试设计 |
| 权限遗漏 | 权限规则未实现 | 权限矩阵验证 |
| 接口不一致 | 前后端对齐缺失 | 接口规格对照 |
