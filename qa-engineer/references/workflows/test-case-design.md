# 测试用例设计流程

**何时使用**: 系统性设计测试用例，将 PRD 验收标准转化为结构化测试场景。

---

## 流程步骤

```
1. 获取输入
   ├── PRD 功能描述 + 业务规则
   ├── 验收标准（Given/When/Then）
   └── 测试计划（测试范围 + 优先级）

2. 分析测试对象
   ├── 提取输入参数
   ├── 提取业务规则
   ├── 提取状态流转
   └── 提取前置条件

3. 选择设计方法
   ├── 边界值分析 → 数值/日期/长度校验
   ├── 状态迁移测试 → 状态流转场景
   ├── 正交实验设计 → 多参数组合
   ├── 等价类划分 → 输入域分类
   └── 错误猜测法 → 异常场景

4. 设计测试用例
   ├── 正常路径用例
   ├── 边界值用例
   ├── 异常路径用例
   └── 状态迁移用例

5. 编写测试用例文档
   └── test-cases-{模块名}-v{版本}.md

6. 覆盖率自检
   ├── 检查 PRD 验收标准覆盖
   ├── 检查业务规则覆盖
   └── 检查边缘场景覆盖
```

---

## 测试设计方法论

### 一、边界值分析

**适用场景**: 数值、日期、长度、数量等有明确边界。

**设计规则**:

| 边界类型 | 测试点 | 示例 |
|---------|-------|------|
| 数值边界 | 最小值-1, 最小值, 最大值, 最大值+1 | 订单金额: 0, 1, 999, 1000, 1001 |
| 长度边界 | 空, 1字符, 最大-1, 最大, 最大+1 | 用户名: "", "a", "19", "20", "21" |
| 数量边界 | 0, 1, 最大-1, 最大 | 购物车商品: 0, 1, 49, 50 |

**用例模板**:

```markdown
### TC-{编号}: {功能}_边界值测试

**测试对象**: {字段/参数}
**边界定义**: 最小={min}, 最大={max}

| 测试点 | 输入值 | 预期结果 | 用例编号 |
|-------|-------|---------|---------|
| 下界外 | {min-1} | 失败/拒绝 | TC-{n}1 |
| 下界点 | {min} | 成功 | TC-{n}2 |
| 上界点 | {max} | 成功 | TC-{n}3 |
| 上界外 | {max+1} | 失败/拒绝 | TC-{n}4 |
```

---

### 二、状态迁移测试

**适用场景**: 状态流转、订单状态机、审批流程。

**设计步骤**:

```
1. 提取状态列表
   例: 订单状态 = [待支付, 已支付, 已发货, 已完成, 已取消]

2. 提取状态迁移规则
   例:
   - 待支付 → 已支付 (支付成功)
   - 待支付 → 已取消 (超时未支付/用户取消)
   - 已支付 → 已发货 (商家发货)
   - 已发货 → 已完成 (用户确认收货)

3. 设计迁移路径测试用例
   - 正常路径: 待支付 → 已支付 → 已发货 → 已完成
   - 异常路径: 待支付 → 已取消
   - 禁止路径: 待支付 → 已发货 (测试非法迁移)
```

**状态迁移矩阵**:

| 当前状态 | 操作 | 目标状态 | 允许 | 用例编号 |
|---------|------|---------|------|---------|
| 待支付 | 支付 | 已支付 | ✅ | TC-S01 |
| 待支付 | 取消 | 已取消 | ✅ | TC-S02 |
| 待支付 | 发货 | 已发货 | ❌ | TC-S03 |
| 已支付 | 发货 | 已发货 | ✅ | TC-S04 |
| 已支付 | 取消 | 已取消 | ❌ | TC-S05 |

---

### 三、正交实验设计

**适用场景**: 多参数组合、配置项组合、筛选条件组合。

**设计步骤**:

```
1. 提取测试因子（参数）
   例: 搜索功能
   - 因子A: 关键词类型（商品名、品牌、SKU）
   - 因子B: 价格范围（无、有区间）
   - 因子C: 分类筛选（无、一级、二级）
   - 因子D: 排序方式（默认、价格升、价格降）

2. 选择正交表
   - 4因子，每因子2-3水平 → L8(2^4) 或 L9(3^4)

3. 生成测试组合
   正交表自动生成 8-9 组合，覆盖所有因子交互
```

**正交表示例 L8(2^4)**:

| 用例编号 | A:关键词 | B:价格 | C:分类 | D:排序 |
|---------|---------|-------|-------|-------|
| TC-O01 | 商品名 | 无 | 无 | 默认 |
| TC-O02 | 商品名 | 有 | 一级 | 价格升 |
| TC-O03 | 品牌 | 无 | 一级 | 价格降 |
| TC-O04 | 品牌 | 有 | 无 | 默认 |
| TC-O05 | SKU | 无 | 二级 | 价格升 |
| TC-O06 | SKU | 有 | 一级 | 价格降 |
| TC-O07 | 商品名 | 有 | 二级 | 默认 |
| TC-O08 | 品牌 | 有 | 二级 | 价格升 |

---

### 四、等价类划分

**适用场景**: 输入域分类、格式校验、类型验证。

**设计规则**:

| 等价类类型 | 定义 | 测试策略 |
|-----------|------|---------|
| 有效等价类 | 符合规则的输入 | 每类取 1 个代表值 |
| 无效等价类 | 不符合规则的输入 | 每类取 1 个代表值 |

**示例**: 手机号输入

| 等价类 | 类型 | 代表值 | 用例编号 |
|-------|------|-------|---------|
| 有效 | 11位数字 | 13800138000 | TC-E01 |
| 无效 | 空值 | "" | TC-E02 |
| 无效 | 少于11位 | 138001 | TC-E03 |
| 无效 | 多于11位 | 138001380000 | TC-E04 |
| 无效 | 含非数字 | 138abc12345 | TC-E05 |
| 无效 | 格式错误 | 12345678901 | TC-E06 |

---

### 五、错误猜测法

**适用场景**: 经验判断易出错点、异常场景补充。

**常见错误点**:

| 场景 | 易错点 | 补充用例 |
|------|-------|---------|
| 数据提交 | 网络中断 | 提交时断网测试 |
| 并发操作 | 重复提交 | 同时点击多次按钮 |
| 数据依赖 | 前置数据不存在 | 引用已删除数据 |
| 权限边界 | 无权限用户访问 | 未登录访问需权限页面 |
| 时间相关 | 跨日期操作 | 跨月/跨年数据统计 |

## PRD → 测试转化规则

### 验收标准转化

**PRD 格式**:

```
Given [前置条件]
When [用户操作]
Then [预期结果]
```

**转化模板**:

```markdown
### TC-{编号}: {场景描述}

**来源**: PRD 验收标准 AC-{编号}

**前置条件**:
- {Given 转化}

**测试步骤**:
1. {When 步骤拆解}

**预期结果**:
- {Then 结果拆解}

**数据准备**:
- {测试数据需求}
```

### 业务规则转化

**PRD 格式**:

```
业务规则 BR-{编号}: {规则描述}
条件: {条件}
结果: {结果}
```

**转化模板**:

```markdown
### TC-{编号}: {规则名称}_规则测试

**来源**: PRD 业务规则 BR-{编号}

**条件组合测试**:
| 条件值 | 预期结果 | 用例编号 |
|-------|---------|---------|
| {满足条件} | {规则结果} | TC-{n}1 |
| {不满足条件} | {非规则结果} | TC-{n}2 |
```

## 用例编号规则

```
TC-{功能编号}{序列号}

功能编号:
├── TC-001~TC-099: 用户管理模块
├── TC-100~TC-199: 商品管理模块
├── TC-200~TC-299: 订单管理模块
└── ...

序列号细分:
├── TC-{n}1~{n}9: 正常路径
├── TC-{n}10~{n}19: 边界值
├── TC-{n}20~{n}29: 异常路径
├── TC-{n}30~{n}39: 状态迁移
└── TC-{n}E{m}: 等价类测试
```
