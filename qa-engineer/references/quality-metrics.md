## 一、质量指标定义

### 1.1 测试通过率

| 指标 | 计算公式 | 目标值 | 说明 |
|------|---------|-------|------|
| 用例通过率 | 通过用例数 / 执行用例数 | ≥ 95% | 核心质量指标 |
| 功能通过率 | 通过功能数 / 测试功能数 | ≥ 90% | 功能维度统计 |
| E2E通过率 | 通过E2E用例数 / E2E用例数 | ≥ 95% | 自动化测试指标 |

### 1.2 缺陷相关指标

| 指标 | 计算公式 | 目标值 | 说明 |
|------|---------|-------|------|
| 缺陷密度 | 缺陷数 / 功能数 | < 0.5 | 每功能平均缺陷 |
| 代码缺陷密度 | 缺陷数 / KLOC | < 5 | 每千行代码缺陷 |
| 缺陷修复率 | 已关闭缺陷 / 总缺陷 | ≥ 90% | 缺陷处理效率 |
| 缺陷重开率 | 重开缺陷数 / 总缺陷 | < 5% | 修复质量指标 |
| 致命缺陷占比 | 致命缺陷数 / 总缺陷 | < 5% | 严重程度分布 |

### 1.3 覆盖率指标

| 指标 | 计算公式 | 目标值 | 说明 |
|------|---------|-------|------|
| 行覆盖率 | 覆盖代码行 / 总代码行 | Service ≥ 80% | 代码覆盖 |
| 分支覆盖率 | 覆盖分支 / 总分支 | Service ≥ 70% | 逻辑覆盖 |
| 功能覆盖率 | 测试功能数 / 总功能数 | P0 ≥ 100% | 需求覆盖 |
| 场景覆盖率 | 测试场景数 / 业务场景数 | ≥ 80% | 业务覆盖 |

### 1.4 效率指标

| 指标 | 计算公式 | 目标值 | 说明 |
|------|---------|-------|------|
| 平均修复时长 | Σ修复时长 / 修复数 | < 2天 | 缺陷处理效率 |
| 平均验证时长 | Σ验证时长 / 验证数 | < 0.5天 | 测试反馈效率 |
| 测试用例密度 | 用例数 / 功能数 | ≥ 5 | 测试深度 |

## 二、指标采集方法

### 2.1 测试通过率采集

```typescript
// 从测试执行结果计算
interface TestExecutionResult {
  totalCases: number;
  executedCases: number;
  passedCases: number;
  failedCases: number;
  blockedCases: number;
}

function calculatePassRate(result: TestExecutionResult): number {
  return result.passedCases / result.executedCases * 100;
}
```

### 2.2 覆盖率采集

```bash
# Jacoco 覆盖率
mvn clean test jacoco:report
# 报告: target/site/jacoco/index.html

# 前端覆盖率
npm run test:coverage
# 报告: coverage/lcov-report/index.html
```

### 2.3 缺陷统计采集

```markdown
# 从 defect-log 统计
总缺陷 = 缺陷清单总数
已关闭 = 状态=关闭的数量
待修复 = 状态=新建/分配的数量
重开 = 状态=重开的数量
```

## 四、质量阈值定义

### 4.1 上线质量门禁

| 指标 | 上线门槛 | 说明 |
|------|---------|------|
| 致命缺陷 | = 0 | 必须全部关闭 |
| 严重缺陷 | ≤ 2 | 允许少量待修复 |
| 测试通过率 | ≥ 95% | 核心功能通过 |
| Service覆盖率 | ≥ 80% | 核心逻辑覆盖 |
| P0功能覆盖 | = 100% | 核心功能必测 |

### 4.2 警告阈值

| 指标 | 警告阈值 | 处理建议 |
|------|---------|---------|
| 测试通过率 | < 95% | 分析失败原因 |
| 缺陷修复率 | < 90% | 加快修复进度 |
| 覆盖率 | < 目标 | 补充测试用例 |
| 缺陷密度 | > 0.5 | 分析模块质量 |

### 4.3 阻塞阈值

| 指标 | 阻塞阈值 | 上线影响 |
|------|---------|---------|
| 致命缺陷 | > 0 | 🔴 阻塞上线 |
| 严重缺陷 | > 5 | 🔴 阻塞上线 |
| P0功能通过率 | < 100% | 🔴 阻塞上线 |
| 测试通过率 | < 80% | 🟠 延后评估 |

## 六、质量改进措施

### 6.1 常见质量问题及改进

| 问题 | 原因分析 | 改进措施 |
|------|---------|---------|
| 通过率低 | 边缘场景未覆盖 | 补充边界值测试 |
| 覆盖率低 | 测试设计不完整 | 增加测试用例密度 |
| 缺陷密度高 | 代码质量差 | 加强 TDD 流程 |
| 缺陷重开多 | 修复质量差 | 增加修复验证 |
| 致命缺陷多 | 核心逻辑缺陷 | 加强核心测试 |

### 6.2 质量改进计划

| 2 | 边界值测试不足 | 边缘缺陷遗漏 | P2 |

## 改进措施

## 七、与测试流程协作

| 测试阶段 | 采集指标 | 使用方式 |
|---------|---------|---------|
| 测试计划 | 用例密度估算 | 资源分配依据 |
| 测试执行 | 通过率实时统计 | 进度监控 |
| 缺陷管理 | 缺陷密度统计 | 质量评估 |
| 测试报告 | 综合指标汇总 | 上线决策 |
| 上线后 | 生产缺陷统计 | 质量复盘 |
