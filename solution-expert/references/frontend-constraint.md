# frontend-constraint

## 规则

### 标准页面模式（直接用于第 4 章描述）

| 适用功能类型 | 页面模式 | 标准描述 |
|---|---|---|
| 数据管理类 | 列表管理页 | 搜索筛选区 + 操作按钮区 + 数据表格（排序/分页）+ 行内操作 |
| 数据新增/编辑 | 详情表单页 | 分区块表单（单列/多列布局）+ 底部操作按钮 |
| 流程审批类 | 审批工单页 | 单据详情 + 审批意见输入 + 流程时间轴（Timeline）|
| 流程汇总入口 | 待办中心页 | 标签页（待办/已办/我发起）+ 列表 |
| 统计报表类 | 统计看板页 | 筛选条件区 + 指标卡（Statistic）+ 图表（ECharts）+ 数据表格 |
| 组织/分类配置 | 树形配置页 | 左侧树形导航（Tree）+ 右侧详情编辑区 |

### 标准工程结构（用于第 4 章应用结构描述）

```
src/
├── views/          按业务域组织，域名称作为子目录
├── components/     跨域复用的公共组件
├── router/         路由配置（动态路由从后端权限数据生成）
├── store/          全局状态（用户信息、权限数据）
├── api/            按业务域组织的接口封装
└── utils/          工具函数
```

### 前后端接口约定（用于第 3、4 章描述）

| 约定项 | 规范 |
|---|---|
| 请求认证 | Authorization: Bearer {token} |
| 统一响应结构 | `{ code, message, data, timestamp }` |
| 错误处理 | 401 跳转登录；403 提示无权限；500 提示系统错误 |
| 接口文档 | Swagger/OpenAPI 自动生成，前后端联调以此为准 |
| 动态路由 | 菜单和路由权限由后端接口返回，前端登录后动态生成 |

### 能力边界（需在方案中单独说明的场景）

| 超出 Ant Design 默认能力的需求 | 需要引入的额外组件 |
|---|---|
| 可视化审批流程设计器 | AntV X6 或 bpmn.js |
| 甘特图 | 专门甘特图库 |
| 拓扑图、关系图 | AntV G6 |
| GIS 地图 | 高德/百度地图 SDK |
| 实时数据刷新（监控大屏）| WebSocket |

遇到以上需求，方案中必须单独说明技术选型，不能当作 Ant Design 的默认能力。
